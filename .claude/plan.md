## 阶段 1: 符号表与类型系统基础
- [ ] 1.1 设计并实现 `Symbol` 类：存储变量名、类型（Type）、是否可变（mut）、是否已赋值、作用域层级
- [ ] 1.2 设计并实现 `SymbolTable` 类：支持嵌套作用域（函数级 + 块级）、插入、查找（当前作用域 + 向上查找）、退出作用域
- [ ] 1.3 扩展现有 `Type` 类：支持 i32、引用类型（&T, &mut T）、数组类型（[T; N]）、元组类型（(T1, T2, ...)）、空元组类型（()）、函数签名类型
- [ ] 1.4 设计并实现 `FunctionInfo` 类：存储函数名、参数列表类型、返回类型

## 阶段 2: 语义分析器框架
- [ ] 2.1 设计 `SemanticAnalyzer` 类：接受 parse tree 的 Node* 根节点，递归遍历进行语义分析
- [ ] 2.2 实现语义错误报告机制：记录错误类型、行号、列号、错误描述，支持多个错误累积输出
- [ ] 2.3 设计遍历策略：对 parse tree 的每个节点类型（Program, Function, Block, Stmt, Expr 等）分发到对应的 visit 方法

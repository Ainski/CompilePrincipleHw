> 本文档为同济大学编译原理课程大作业 1 的原始需求说明，记录了词法规则、语法规则、评分标准和提交要求等。内容源自课程 PPT。

## 1 项目概述

- **目标**：了解类 Rust 词法、语法规则，对类 Rust 示例程序实现词法分析和语法分析，并输出分析结果。
- **扩展功能**：不限。
- **分组**：每组 2–3 人。
- **产出**：程序源代码与可执行文件、设计文档、程序实例与运行结果截屏、报告 PPT。

### 类 Rust 语言背景

Rust 是一门能够安全高效地编写系统级程序的语言，在内存安全方面具有显著优势：

| 特性 | 说明 |
|------|------|
| 性能 | 速度极快，性能比肩 C 语言，内存效率极高，无需运行时或垃圾收集器 |
| 可靠性 | 丰富的类型系统和所有权模型保证了内存安全和线程安全 |
| 生态性 | 出色的文档、友好的编译器错误提示、集成的包管理器和构建工具 |
| 社区 | 自 2015 年以来，连续多年为 Stack Overflow 调查中开发者最爱的编程语言 |

### 2.2 标识符

```
(字母|_)(字母|数字|_)*
```

### 2.3 数值

```

### 2.4 赋值号

```

### 2.5 算术与关系运算符

```
+ | - | * | / | == | > | >= | < | <= | != | &
```

### 2.6 界符

```
( | ) | { | } | [ | ]
```

### 2.7 分隔符

```
; | : | ,

```

### 2.9 注释号

```

```

### 2.10 字母与数字

- 字母：`a`–`z`、`A`–`Z`
- 数字：`0`–`9`

### 2.11 结束符

```

```

### 2.12 词法分析要求

- `if123` 应识别为**标识符**（一整个单词）。

## 附录：规则间的羁绊与思考

### 规则间的交互约束

- **2.2 赋值语句 + 拓展 `<类型>` 的规则（如 8.1）**：类型多于一种，变量赋值时需要检测类型是否对应，不对应应报错。
- **5.2 for 循环结构 + 8.2 数组**：数组是可迭代结构，可被 for 循环访问；其他类型的变量不可迭代，被 for 循环访问时应报错。
- **2.2 赋值语句 + 6.1 变量不可变属性 + 循环的规则（如 5.2）**：若循环中存在对不可变变量进行赋值的操作，应报错。
